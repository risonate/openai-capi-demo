OpenAI Pixel · Client-Side Reference

The browser-side half of the CAPI integration. Pairs with send-events.js.

How to use this page: open browser DevTools (Cmd+Option+I on Mac), go to the Network tab, filter for bzr. Click a button below to fire an event. You'll see a request to bzr.openai.com/v1/events (or bzrcdn.openai.com for the SDK itself). With the fake pixel ID, it'll 404 — but the call shape and headers are real.

The init snippet (already running)

Right now, this page's <head> has the snippet that loaded the SDK. Check the Network tab — you should see a request to bzrcdn.openai.com/sdk/oaiq.min.js already completed.

The init snippet — copy this into your site's template:

<script>
  (function (w, d, s, u) {
    if (w.oaiq) return;
    var q = w.oaiq = function () { q.q.push(arguments); };
    q.q = [];
    var js = d.createElement(s); js.async = true; js.src = u;
    var f = d.getElementsByTagName(s)[0];
    f.parentNode.insertBefore(js, f);
  })(window, document, "script", "https://bzrcdn.openai.com/sdk/oaiq.min.js");

  oaiq("init", { pixelId: "<YOUR-PIXEL-ID>" });
</script>

Firing events

Once init has run, the advertiser fires events with oaiq("measure", <event_type>, <event_props>, <event_options>). Click any button to fire a sample event and watch the Network tab.

Click a button to fire an event. Output will appear here, and the request will show in DevTools → Network.

The dedup contract

The critical field in the order_created event is event_id in the third argument. This must match the id field that your server-side CAPI call (send-events.js) sends for the same conversion. OpenAI dedupes by matching these values — same identifier flows through both pixel and CAPI, OpenAI counts the conversion once.

Use the advertiser's authoritative order ID — Shopify order number, Stripe PaymentIntent ID, Salesforce opportunity ID. Whatever value the advertiser already considers their source of truth. Don't generate a UUID in the pixel that the server can't see.

What this looks like with a real ChatGPT click

When a user clicks an OpenAI ad and lands here, the URL would look like:

https://shop.example.com/checkout/success?oppref=gAAAAABqCQzH...&utm_source=chatgpt&utm_medium=cpc&...

The pixel's SDK automatically captures oppref from the URL and stores it in a first-party cookie called __oppref. Any subsequent oaiq("measure", ...) calls in that user's session will include the oppref automatically. This is the one thing CAPI doesn't do — server-side, you have to capture oppref yourself from the inbound URL and replay it on every event.
